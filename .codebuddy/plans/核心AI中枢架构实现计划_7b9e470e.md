## 用户需求

1. 核心AI（Central AI）- 每个公司一个，负责存储和处理所有业务数据
2. 普通员工（真实用户）- 通过核心AI连接才能访问公司数据
3. 虚拟AI员工 - 也是通过核心AI连接访问数据
4. 所有交互都通过核心AI，数据不直接暴露给任何人
5. 真实员工可以请求虚拟AI员工处理任务（如"帮我写产品介绍"）
6. 虚拟AI员工之间也可以协作

## 核心功能

- 核心AI服务层：处理所有请求的中枢，负责数据存储、权限控制、任务分配
- Actor模型扩展：支持真实员工和虚拟AI员工的统一管理
- 交互API：支持对话、任务请求、AI员工协作
- 虚拟AI员工管理系统：定义和管理虚拟AI员工角色
- 交互记录：完整的对话和任务协作历史

## 技术栈

- Backend: Python FastAPI + SQLAlchemy + PostgreSQL
- AI: DeepSeek API (已配置)
- 向量存储: pgvector (用于语义搜索)
- 任务队列: 异步任务处理

## 架构设计

### 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     核心AI (Central AI)                        │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │
│  │  意图理解    │  │  任务生成    │  │  权限控制    │             │
│  │ (AIService) │  │ (TaskEngine)│  │ (AuthCore)  │             │
│  └─────────────┘  └─────────────┘  └─────────────┘             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │
│  │  数据存储    │  │  交互记录    │  │  贡献计算    │             │
│  │ (EventStore)│  │ (ChatLog)   │  │ (Evaluation)│             │
│  └─────────────┘  └─────────────┘  └─────────────┘             │
└─────────────────────────────────────────────────────────────────┘
                            ▲
                            │ 统一API入口
          ┌─────────────────┼─────────────────┐
          ▼                 ▼                 ▼
   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
   │  真实员工    │   │ 虚拟AI员工   │   │  外部系统    │
   │ (HumanActor)│   │ (AIAgent)   │   │ (External)  │
   └─────────────┘   └─────────────┘   └─────────────┘
```

### Actor类型

- HUMAN: 真实员工
- AI_AGENT: 虚拟AI员工
- EXTERNAL: 外部合作者

### 核心API设计

```python
# 统一交互入口
POST /api/central/input     # 任何输入都通过这里
POST /api/central/query     # 自然语言查询
POST /api/central/chat      # 与AI对话/请求任务

# Actor管理
GET  /api/actors            # 获取所有Actor
GET  /api/actors/me        # 当前用户信息
POST /api/ai-agents        # 创建虚拟AI员工

# 交互历史
GET /api/central/history   # 获取交互历史
```
